### 4. You mentioned CRDT-based real-time collaboration—why CRDT over WebSockets + server sync?

**Answer:**  
CRDTs eliminate the need for a central conflict resolution authority. In collaborative environments, especially with intermittent connectivity, CRDT ensures eventual consistency without data loss. - State synchronization across clients
- Handling presence (who’s online, cursor positions)

- Efficient diff-based rendering

## 9) “You had an infinite OSRM request issue earlier. What caused it, and how did you fix it?”

**You:**  
The root cause was repeated triggers requesting the same route while a request was already in-flight—essentially a feedback loop. The fix is in useDirections: it tracks the current route key, dedupes requests for the same key, and uses AbortController to cancel the previous request when the key changes. It also caches by (branchId + origin) so revisiting doesn’t refetch. That turns a potentially runaway loop into a stable, deterministic state machine.
